CalciumInsight for Calcium Transient Analysis

This guide explains how to use CalciumInsight to load calcium imaging traces, select a region of interest, smooth the signal using FFT low-pass filtering, detect calcium transients, compute transient-level and trace-level metrics, evaluate FFT smoothing settings, and export publication-ready results.

Purpose

1. Overview

CalciumInsight is an interactive Shiny application designed to analyze calcium transient traces from one or more regions of interest (ROIs). The app first smooths the selected trace using a low-pass Fast Fourier Transform (FFT) reconstruction and then detects transient peaks using user-defined peak criteria. The app reports quantitative metrics for each transient and summary metrics for the complete trace.

The central idea is to preserve biologically meaningful low-frequency calcium transient dynamics while reducing high-frequency noise. The resulting smoothed signal is used for peak detection, peak annotation, amplitude estimation, prominence calculation, width measurements, rise-time analysis, and optional area-under-the-curve analysis.

What the app does

  • Loads example data or user-provided CSV, TSV, or JSON files.
  • Selects a single ROI trace for detailed analysis.
  • Applies FFT low-pass smoothing with a user-controlled frequency fraction.
  • Detects peaks using ascent, descent, height, distance, FWHP, and prominence thresholds.
  • Supports multiple baseline definitions, including dynamic percentile baselines.
  • Calculates transient-level and trace-level calcium metrics.
  • Provides Fourier analysis plots, a frequency table, and an explicit reconstruction formula.
  • Exports CSV tables and a high-resolution calcium trace graph.

Scope

2. Intended use

The application is intended for users working with calcium fluorescence traces, such as ΔF/F0 time series extracted from microscopy or calcium imaging pipelines. It is especially useful when the user needs a reproducible and visual workflow for comparing raw and smoothed traces, identifying calcium transient events, and exporting standardized quantitative descriptors.

Important: The app is an analytical and visualization tool. Final biological interpretation should consider experimental design, acquisition frequency, noise level, cell type, stimulation protocol, and the appropriateness of the selected peak thresholds.

Input

3. Data input

Accepted file formats

The app can be used with the built-in example dataset or with a user-uploaded file. Supported user-uploaded formats are:

CSV

Comma-separated text file. Recommended for most users.

TSV

Tab-separated text file. Useful when decimal or comma conventions conflict.

Required table structure for CSV/TSV files

The first column must contain time values. Each additional column must correspond to one ROI. The app assumes that the ROI number selected in the interface refers to the ROI columns after the time column. For example, ROI 1 corresponds to the second column of the dataset.

Time ROI1 ROI2 ROI3

0.50.5360.4470.619

1.50.5490.4750.632
Data preparation recommendation: Check that the time column is numeric, the ROI columns are numeric, and missing values have been handled before upload. The time unit used in the first column will also define the time unit of all reported time metrics.

Basic operation

4. Quick start

  1. Choose the data source. Select Example Data = Yes to test the app with built-in calcium-like traces. Select Example Data = No to upload your own CSV, TSV, or JSON file.
  2. Select the ROI. Enter the number of the region of interest in Region of Interest (ROI). ROI 1 corresponds to the first signal column after the time column.
  3. Choose the FFT smoothing intensity. Use FFT Low-Frequency Fraction to control how much of the low-frequency content is retained. Smaller values produce stronger smoothing; values closer to 1 preserve more of the original signal.
  4. Keep or remove the mean. Leave Keep mean (DC component) checked when the global signal level should be preserved during FFT reconstruction.
  5. Set peak detection parameters. Adjust peak height, ascent, descent, minimum distance, FWHP threshold, and prominence threshold until the detected peaks match the expected calcium transients.
  6. Select a baseline definition. Choose a fixed or dynamic baseline depending on the experimental signal behavior.
  7. Inspect outputs. Review the SummaryData, Peaks, Fourier analysis, and FFT selection criteria tabs.
  8. Export results. Download transient metrics, trace metrics, or a high-resolution calcium trace graph.

Analysis logic

5. Analysis workflow

1

Load data

The app reads the dataset and displays the number of ROIs and time observations.

2

Select ROI

The selected ROI trace is extracted as a two-column time-signal dataset.

3

FFT smoothing

The signal is transformed with FFT, low-frequency components are retained, and the smoothed trace is reconstructed.

4

Peak detection

Peaks are detected on the smoothed trace using user-defined peak parameters.

5

Baseline correction

The selected baseline is used to interpret amplitude, AUC, and FWHM-related quantities.

6

Metric extraction

The app computes transient-level and trace-level summaries and makes them available for export.

Controls

6. Parameter guide

Example Data
When set to Yes, the app loads a built-in dataset with four simulated ROI traces. This is useful for learning the workflow.
Dataset
File upload field for CSV, TSV, or JSON files when Example Data is set to No.
Region of Interest (ROI)
Numeric selector indicating which ROI trace should be analyzed. The first ROI is the first signal column after the time column.
FFT Low-Frequency Fraction
Controls the fraction of positive low-frequency components retained during reconstruction. A smaller fraction removes more high-frequency variation and produces stronger smoothing. A larger fraction preserves more details and may retain more noise.
Keep mean (DC component)
Keeps the global average of the trace in the reconstructed signal. This option is recommended unless the analysis intentionally requires removing the global offset.
Peak Height (min)
Minimum signal height required for a point to be considered a peak.
Peak Ascent
Minimum number of consecutive increasing points before the peak.
Peak Descent
Minimum number of consecutive decreasing points after the peak.
Min Peak Distance
Minimum separation between detected peaks, expressed in number of observations.
FWHP (min)
Minimum full width at half prominence. Peaks below this width are filtered out.
Prominence (min)
Minimum vertical prominence of the peak above its preceding local minimum.
Area Under the Curve (AUC)
When set to Yes, the app calculates and visualizes the area above the selected baseline.
Raw Data
When set to Yes, the raw trace is overlaid on the calcium trace graph for comparison with the smoothed trace.
Full Width at Half Maximum
When set to Yes, FWHM segments are displayed on the calcium trace graph.

Baseline correction

7. Baseline options

Baseline selection influences amplitude correction, AUC calculation, and FWHM estimation. The best option depends on whether the trace has a stable baseline, a drifting baseline, or long-term fluctuations.

Option in app How it is calculated When to use it
Reference Level 0 Uses 0 as the baseline. Use when traces are already normalized around zero or when 0 is the experimental reference.
Standard definition Uses the mean signal before the first detected transient onset. Use when the initial segment represents resting fluorescence.
Interval Uses the mean signal between a user-defined lower and upper time limit. Use when a known baseline interval exists.
Your baseline Uses a manually entered numeric value. Use when the baseline has been estimated externally or must be fixed across ROIs.
Min Uses the minimum value of the smoothed signal. Use cautiously for clean traces where the minimum approximates baseline.
Rolling percentile baseline Calculates a percentile baseline using a centered moving window at each time point. Use for traces with slow baseline drift.
Fixed-window percentile baseline Divides the trace into non-overlapping windows and assigns one percentile baseline per window. Use when the baseline changes over time but should remain constant within each window.
Window size: For rolling and fixed-window percentile baselines, the window size must use the same time unit as the dataset time column. For example, if time is measured in seconds, 10 minutes should be entered as 600 seconds.

Smoothing selection

8. Selecting the FFT Low-Frequency Fraction

The FFT selection criteria panel evaluates candidate low-frequency fractions: 0.10, 0.15, 0.20, 0.25, 0.30, 0.35, and 0.40. This panel helps users select a smoothing level that balances noise reduction with preservation of calcium transient morphology.

1. Signal morphology

Checks whether the smoothed signal remains strongly aligned with the raw trace.

2. Residual behavior

Assesses the magnitude and structure of the residual signal after smoothing.

3. Spectral power

Evaluates the fraction of periodogram power retained by the selected low-frequency components.

4. Mean peak stability

Compares each candidate mean peak amplitude to a weighted reference mean across candidates.

The app can recommend a candidate fraction based on these criteria. However, the recommendation is intended as a quantitative guide rather than an automatic decision. Users should inspect the raw/smoothed overlay, residual plot, periodogram, and detected peaks before finalizing the value.

Results

9. Output tabs and interpretation

SummaryData

Displays a dataset summary with the number of ROIs and the number of time observations. It also shows the uploaded or example dataset in an interactive table.

Peaks → Metrics

Contains two tables: transient metrics and trace metrics. Transient metrics describe each detected event. Trace metrics summarize the entire selected ROI trace.

Peaks → Metric plots

Shows the calcium trace with detected peaks, baseline references, prominence/FWHP/FWHM annotations when selected, the first derivative plot, and the raw-versus-smoothed comparison.

Peaks → Fourier analysis

Reports the FFT summary, smoothed fit plot, residual plot, spectrum, frequency table, and explicit reconstruction formula using the retained FFT frequencies.

FFT selection criteria

Compares candidate FFT fractions using morphology, residual, spectral power, and mean-peak criteria. This tab is useful for justifying the selected smoothing parameter.

Export

10. Downloadable files

Trace Metrics

Downloads a CSV file with peak-level measurements, including amplitude, peak occurrence time, prominence, FWHP, FWHM, rise time, transient onset, and rise rate.

Transient Metrics

Downloads a CSV file with trace-level summary metrics such as time onset, frequency, baseline, number of peaks, and AUC when enabled.

Calcium Trace Graph

Downloads the annotated calcium trace as a high-resolution PNG image suitable for reports, presentations, or manuscript preparation.

Recommendations

11. Recommended analysis practices

  • Start with the example dataset to understand the interface before uploading your own data.
  • Inspect the raw and smoothed traces together before accepting a smoothing value.
  • Use the FFT selection panel to document why a given low-frequency fraction was selected.
  • Adjust peak height and prominence thresholds together to reduce false-positive peak detection.
  • Use a dynamic baseline when the trace shows slow baseline drift over time.
  • Keep all parameter settings consistent when comparing ROIs, groups, or experimental conditions.
  • Export both the metrics and the annotated graph to preserve a reproducible analysis record.

Support

12. Troubleshooting

Reduce the minimum peak height, reduce the minimum prominence, reduce the minimum peak distance, or check whether the FFT fraction is too small and oversmoothing the trace.

Increase the minimum peak height, increase the minimum prominence, increase the minimum peak distance, or reduce high-frequency noise using a smaller FFT low-frequency fraction.

Increase the FFT low-frequency fraction to retain more frequency components. Also verify that Keep mean (DC component) is enabled when the global fluorescence level should be preserved.

Try a different baseline mode. For drifting traces, use a rolling percentile or fixed-window percentile baseline. For stable traces, use the standard definition or a known interval.

Confirm that the first column is time, all ROI columns are numeric, and the file format is CSV, TSV, or JSON. If the file is large, reduce unnecessary columns or split the dataset by experiment.

Questions

13. Frequently asked questions

Does the app analyze all ROIs at once?

The interface is designed for detailed inspection of one selected ROI at a time. The user can change the ROI number to repeat the workflow for another trace.

What does a smaller FFT fraction mean?

A smaller fraction keeps fewer low-frequency components. This increases smoothing but may remove biologically meaningful details if set too low.

Should the mean component be kept?

In most calcium trace analyses, yes. Keeping the DC component preserves the global average level of the signal after FFT reconstruction.

What is the difference between FWHP and FWHM?

FWHP is based on half prominence, using the vertical distance between the peak and a local minimum. FWHM is based on half maximum relative to the selected baseline.

Can I use AUC with a dynamic baseline?

Yes. When a rolling or fixed-window percentile baseline is selected, the app uses the corresponding baseline trace to estimate the positive area above baseline.

How should I report the analysis?

Report the file format, ROI analyzed, FFT low-frequency fraction, whether the DC component was retained, peak detection thresholds, baseline mode, and exported metrics.

Suggested methods description

Calcium traces were smoothed using a Fast Fourier Transform low-pass reconstruction. For each selected ROI, the discrete Fourier transform of the signal was computed, a user-defined fraction of the lowest positive-frequency components was retained, and the inverse transform was used to reconstruct the denoised signal. Peaks were detected on the smoothed trace using user-defined ascent, descent, minimum height, minimum distance, minimum prominence, and width criteria. Transient-level metrics included amplitude, peak occurrence time, prominence, FWHP, FWHM, peak rise time, transient onset time, and rise rate. Trace-level metrics included onset time, peak frequency, baseline, number of detected peaks, and area under the curve when enabled.
